# 新版排課系統（課務關聯版）

這份修改以現有最新版為基礎，將音教雲的排課操作重整成可讀性較高的新版介面；保留舊音教雲入口，方便逐項核對。

## 本版重點

- 日表以教室為欄、30 分鐘為一格；空白格可新增固定課、單堂課、體驗課或教室租用。
- 固定課可每週或隔週延續；複製、剪下與貼上調課都會重新檢查衝突。
- 顏色區分固定課、調課／單堂、租用、體驗、請假及已完成狀態。
- 同時檢查教室、老師與學生的時間衝突，以及課程跨越的每一個教室時段規則。
- 教室可依星期與每 30 分鐘設定：禁止排課、禁止租用、允許科目。
- 科目會篩選可授課老師；老師資料可設定多個可教授科目。
- 同一學生可保留多科目、多期學費；每一期保存當時的收費方案、金額、堂數、抽成與請假扣堂規則快照。
- 繳費、退款、簽到、請假、缺席與排課分開留帳；只有有效出席／缺席或方案指定要扣堂的請假才扣堂，而且同一堂只扣一次。
- 已有簽到紀錄的課程不可直接剪下或取消，避免堂數與歷史帳被破壞。
- 字體、按鈕、格線與視覺層級已放大，適合一般電腦螢幕操作。

## 資料範圍

後端使用明確的「課務白名單」，只讀學生、學生學費與付款／退款、老師、教室、科目、收費方案、固定課、單堂／體驗課、簽到、請假、缺席、老師獎勵／扣薪與教室租用。

商品、庫存、銷售、進貨、供應商、倉庫與平台訂單已明確排除，不會進入新版排課模組。

移轉資料載入後仍是唯讀核對模式，不會覆蓋音教雲來源或 `opsInjiaoyunTest` 既有資料。畫面中的示範操作只存在瀏覽器本機。

## 安裝順序

1. 將 ZIP 內檔案依原有資料夾位置覆蓋到最新版專案。
2. 部署課務預覽函式：`firebase deploy --only functions:loadInjiaoyunEducationPreview`
3. 將網頁檔案照目前 GitHub Pages 流程發布。
4. 從營運中心進入「課程日表」，可選擇舊音教雲或新版排課系統。
5. 在新版按「載入已移轉課務資料」，輸入既有手動同步密碼後逐項核對。

本版沿用既有 `INJIAOYUN_MANUAL_SYNC_PIN` Secret，不需把密碼寫進程式或 GitHub，也不需修改 Firestore Rules。
