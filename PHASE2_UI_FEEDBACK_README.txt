第二階段 UI 進度提示統一修正

目的：
- 將第一階段以外的頁面也接上同一套綠底白字進度提示。
- 按下需要等待的按鈕時，按鈕本身會顯示「儲存中／送出中／查詢中／核准中／檔案產生中」等進度文字。
- 成功或失敗訊息會顯示在按鈕下方，不再讓使用者滑到頁面上方找訊息。

這次主要涵蓋：
- 公告、任務、例行工作、教育訓練
- 老師拿貨與詢價
- 在職證明、教學證明、證明審核與紀錄
- 應聘管理、資料異動審核
- 薪資、班表、資料中心
- 補打卡／臨時出勤／請假管理
- 通知設定、LINE / Email 檢查
- 外聘老師管理、契約條文、薪資資料
- 估價單、點數卡、密碼更新

沒有修改：
- Firestore 資料表名稱
- Firebase Functions
- LINE / Email 發送規則
- 租賃契約邏輯
- 員工資料結構

使用方式：
- 將壓縮檔內的檔案覆蓋到 GitHub Pages 的 play-card 根目錄。
- 必須一起上傳 ui-action-feedback.js。
- 本次不需要部署 Firebase Functions。
