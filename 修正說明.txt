【這次修正重點】
1. 不再讓 <audio> 直接吃 Google Drive 網址。
2. 改成前端先按「載入錄音」，再向 Apps Script 讀取 Drive 音檔內容。
3. Apps Script 會把 Drive 裡的音檔轉成 data URL 回給前端播放。
4. 這樣就不依賴 Google Drive 的直接串流行為，比較能避開 0:00 / 0:00 與按鈕不能按的問題。

【要更新的檔案】
- GitHub：app.js、task.html
- Apps Script：apps-script.gs（重新部署）

【測試方式】
1. 換掉 GitHub 的 app.js 與 task.html
2. Apps Script 貼上新的 apps-script.gs 並重新部署 Web App
3. 強制重新整理 task.html
4. 到任務卡點「載入錄音」

【判讀結果】
- 若載入後能播：代表原本問題是 Google Drive 直接串流不穩。
- 若載入後仍失敗，而且顯示錄音檔是空的：代表那一筆錄音本身壞掉。
- 若新錄音可播、舊錄音不能播：代表舊資料格式異常，不是播放器介面問題。
