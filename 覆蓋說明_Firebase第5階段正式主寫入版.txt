Firebase 第5階段：正式主寫入 Firebase

覆蓋方式：
1. 解壓縮 ZIP。
2. 將 ZIP 內所有檔案丟到 GitHub 專案根目錄。
3. 同名檔案全部選「取代」。
4. Commit changes。
5. GitHub Pages 更新後，每頁 Ctrl + F5 測試。

這版做的事：
- 登入、註冊、打卡、請假、工讀、打卡修正、通知設定：優先直接寫 Firebase。
- 多數管理資料：優先寫 Firebase collection。
- Email、LINE、PDF、Excel、雲端檔案上傳等仍保留原本 GS/外部服務流程。

重要提醒：
- 這是主寫入切換版，請先用 1 位測試員工帳號測打卡/請假/工讀/管理審核。
- 目前 Firestore 規則仍是測試公開版，確認功能後要再鎖權限。
